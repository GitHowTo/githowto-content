### Цілі

- Навчитися працювати з декількома гілками з різними (і, можливо, конфліктуючими) змінами.

Як я вже згадував раніше, Git дозволяє працювати з декількома гілками одночасно. Це дуже корисно при роботі в команді, оскільки люди можуть працювати над різними функціями паралельно. Це також корисно, коли ви працюєте наодинці: розробляючи функції в окремих гілках, ви все одно можете виправляти помилки і випускати незначні оновлення, використовуючи стабільний код в гілці `main`.

## _01_ Створіть файл `README`

Створимо файл `README`. Він буде пояснювати, про що наш проєкт.

<File file="README">
```
This is the Hello World example from the GitHowTo tutorial.
```
</File>

## _02_ Закомітьте файл `README` у гілку `main`

Наразі ми перебуваємо у гілці `style`. Файл `README` не є частиною цієї гілки, тому перед комітом слід перейти до гілки `main`.

<Run>
```
git switch main
git add README
git commit -m "Added README"
```
</Run>
