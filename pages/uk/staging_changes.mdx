### Цілі

- Навчитися індексувати зміни для наступних комітів.

## _01_ Додайте зміни

Зараз, дайте команду Git проіндексувати зміни. Перевірте стан:

<Run>
```
git add hello.html
git status
```
</Run>

Ви побачите:

<Output>
```git
==include[output/05-01.txt]
```
</Output>

Зміни файлу `hello.html` було проіндексовано. Це означає, що Git тепер знає про зміни, але вони поки що не _перманентно_ (читай, _назавжди_) записані до репозиторія. Наступний коміт буде містити проіндексовані зміни.

Якщо ви вирішили, що _не_ хочете комітити зміни, команда стану нагадає вам про те, що за допомогою команди `git restore --staged` можна зняти індексацію цих змін.
